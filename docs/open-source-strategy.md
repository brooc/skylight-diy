# Open Source Strategy

## Goal

Make the project easy for families, makers, and developers to try, fork, and improve.

## Positioning

A hackable family command center for tablets and wall displays.

## Naming note

The project now uses Daymark as its distinct public name after starting from a descriptive working name. Public promotion should use Daymark branding to avoid trademark confusion and make the project stand on its own.

## Suggested license

MIT or Apache-2.0 are good candidates.

MIT is simple and permissive. Apache-2.0 adds an explicit patent grant. We should choose deliberately before public launch.

## Repo hygiene

Recommended initial files:

- README.md
- LICENSE
- CONTRIBUTING.md
- CODE_OF_CONDUCT.md
- SECURITY.md
- docs/
- .github/ISSUE_TEMPLATE/
- .github/PULL_REQUEST_TEMPLATE.md

## Contribution areas

- Tablet UI
- Calendar integrations
- Chore/reward workflows
- Meal planning
- Magic Import extraction
- Fire Tablet setup/testing
- Docker/self-hosting
- Accessibility
