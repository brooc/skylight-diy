# v0.1 Operational Checklist

This checklist is the practical runbook for validating and operating the v0.1 build.

## Smoke test checklist

- [ ] `docker compose up --build app` starts API and web successfully.
- [ ] `http://localhost:3000/api/health` returns `ok`.
- [ ] `http://localhost:5173` loads and routes to setup or today.
- [ ] Settings lock/unlock works with local admin PIN.
- [ ] Chores page can add a task and toggle completion.
- [ ] Meals page can add a meal entry.
- [ ] Lists page can add a list and toggle list items.
- [ ] Calendar Today and Week pages render events.

## Automated regression suite

- [x] API integration suite covers setup/session, chores/rewards, meals/lists, and calendar/OAuth degraded behavior.
- [x] Web suite covers Today add-menu behavior, Today refresh/degraded banner rendering, Google Calendar settings OAuth-state/source-edit UX, and `?add=1` quick-add page states.
- [x] Run all tests in Docker with `docker compose exec app pnpm test`.
- [x] Run coverage in Docker with `docker compose exec app pnpm test:coverage`.
- [ ] Prefer no-mock integration and browser tests for future coverage increases.

## Calendar integration checklist

- [ ] Follow [Google Calendar Setup](google-calendar-setup.md).
- [ ] Connect Google Calendar from Settings.
- [ ] Import calendar sources.
- [ ] Rename/recolor imported sources and assign at least one source to a household person.
- [ ] Ensure `GET /api/calendar/events` returns events for the selected range.
- [ ] Confirm Week view refresh button updates status badge.
- [ ] Confirm degraded warning appears when provider fetch fails.

## Fire Tablet soak test (2-4 hours)

- [ ] Open app in Silk Browser in landscape mode.
- [ ] Keep screen active with standard kiosk/browser settings.
- [ ] Verify no recurring white screens or blank calendar states.
- [ ] Verify touch remains responsive on navigation and checklist toggles.
- [ ] Verify readability from a few feet away in ambient daylight.
- [ ] Verify tablet remains thermally stable while charging.

## Backup and restore basics

Backup Postgres data:

```bash
docker exec skylight-diy-postgres pg_dump -U skylight skylight_diy > skylight_diy_backup.sql
```

Restore Postgres data:

```bash
cat skylight_diy_backup.sql | docker exec -i skylight-diy-postgres psql -U skylight skylight_diy
```

Important notes:

- Keep `.env` backed up with database dumps.
- `TOKEN_ENCRYPTION_KEY` must be preserved to decrypt stored Google tokens.
- If encryption key is lost, reconnect Google accounts after restore.
- Restore into a fresh database unless you intentionally handle merge conflicts.

## Known limitations (v0.1)

- Google OAuth and calendar read paths are functional but still early-stage.
- Calendar write/create/update flows are not implemented.
- No automated Playwright E2E suite is included yet (Vitest API/web regression suite is now in place).
- No background worker for periodic sync is included yet.
