# Security

This project may handle sensitive household data including calendars, schedules, documents, and imported text.

## Principles

- Minimize stored sensitive data.
- Do not log raw imports by default.
- Require review before creating calendar events from extraction output.
- Store credentials securely.
- Prefer self-hostable architecture.

## Reporting

Until the project is public and maintainers are defined, please avoid sharing sensitive vulnerabilities publicly.
